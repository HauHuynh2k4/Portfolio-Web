STOMS

Centralized Operations Management System · Capstone Project SEP490 · FPT University

STOMS is a full-scale platform for managing student organization operations, built by a team of 5 as a capstone project. The system covers member management, training sessions, events, equipment, finance, and real-time notifications in a single integrated platform.

My role: Back-End Developer & Team Leader

Led a team of 5 developers. Personally responsible for the following modules:

  • Face Recognition Attendance — integrated AWS Rekognition to identify members from group photos using a greedy face-matching algorithm; supports bulk check-in/check-out
  • Attendance Management — full CRUD for attendance records, session-based tracking, check-in/out timestamps with image evidence
  • Member Assignment — task assignment system with approval workflows, status tracking, and task reports
  • Finance / Fund Management — member wallets, contributions, expenses, borrowings, and full transaction history with audit logging

Key technical achievements:

  • Designed and implemented RESTful APIs with ASP.NET Core following clean architecture (API → Service → Repository)
  • Integrated AWS Rekognition for real-time face detection and comparison across group photos
  • Built real-time notification system using SignalR WebSocket hub
  • Implemented automated background jobs with Hangfire (session status updates, daily reminders)
  • Containerized the application with Docker and deployed to Render
  • Designed PostgreSQL schema for 40+ entities with Entity Framework Core
