ASA Platform

SaaS Platform · Startup Project EXE201 · FPT University

ASA is a SaaS platform built for the EXE201 entrepreneurship course, developed by a team of 4 over 3 months and deployed to production. The system consists of two independent back-ends: the Tenant BE powers each shop with a full suite of business tools, while the Platform BE gives the company control over tenant subscriptions and revenue. Payment is handled via SePay with VietQR code generation and webhook-based order finalization.

My role: Back-End Developer

Worked across both services. Responsible for the following modules:

  • AI Chatbot (Tenant) — built a shop-scoped AI advisor that answers sales questions and surfaces analytics (revenue, inventory, customers, product performance) via dedicated endpoints
  • Product Management (Tenant) — full CRUD with image upload, unit management, category, and status toggling; feature-gated via custom [RequireFeature] attribute
  • Vouchers & Promotions (Tenant) — voucher creation and validation, promotion programs with product-level discount rules
  • Order Pricing Flow (Tenant) — pricing calculation logic that applies promotions, vouchers, and rank-based discounts before finalizing an order
  • SePay Webhook (Tenant) — webhook handler that confirms payment, deducts inventory, calculates profit, and triggers real-time notifications via SignalR and FCM
  • Revenue Reports (Tenant) — statistics overview, shift-close report, and professional Excel export
  • Tenant Onboarding (Platform) — shop creation and subscription provisioning flow
  • Promotions & Revenue Reports (Platform) — platform-level promotion management and revenue analytics (monthly revenue, shop-by-package, new vs renewal stats)
  • AI Chatbot (Platform) — company-side chatbot for internal shop data queries

Key achievements and learnings:

  • Shipped a production-ready SaaS product with two independent back-ends within 3 months as part of a startup course
  • Learned and integrated SePay payment gateway — handling VietQR generation, webhook verification, and idempotent transaction processing
  • Built a feature-based authorization system using a custom [RequireFeature] attribute to gate endpoints by subscription plan
  • Integrated an AI chatbot backed by real shop data, providing actionable sales and inventory insights
  • Gained experience designing a dual-service architecture where Platform and Tenant operate independently with separate databases
  • Learned how to scope and prioritize features under startup constraints — balancing speed of delivery with code quality

Ice Quest

2D Snowboarding Game · PRU212 · FPT University

A 2D physics-based snowboarding game built with Unity and C# as a lab assignment for PRU212 at FPT University, developed by a team of 5. Players race down icy slopes across 3 progressively harder levels, collecting coins, picking up power-ups, performing aerial spins for bonus score, and avoiding obstacles to reach the finish line. The game features a star rating system, per-level high score persistence, and level unlock progression.

My role: Game Developer

Responsible for the following systems:

  • Player Controller — physics-based slope movement using Rigidbody2D with speed and torque scaling by terrain angle; balance and stability systems; jump with aerial rotation via A/D keys
  • Scoring System — distance-based score (+1 per unit traveled) and spin bonus tracking (full 360 rotations while airborne; +100 for first spin, +150 per additional spin)
  • Spin Detection — detects full rotations while airborne (minimum 0.5s in air) and awards bonus score on landing with a popup display
  • UI System — built all UI panels and interactive elements: Main Menu, Level Select, Pause Panel, Game Over Panel, Win Panel, Settings Panel, HUD bars (health, score, speed display), and all buttons
  • Score Persistence — per-level high score saved to JSON (desktop) or PlayerPrefs (WebGL); level completion status persisted for unlock checks and star display
  • Map Design — contributed to designing and laying out the level terrain

Key achievements and learnings:

  • Delivered a complete 3-level game with physics, power-ups, scoring, and persistence as a solo project
  • Learned Unity 2D physics in depth — Rigidbody2D, PhysicsMaterial2D, slope-based force application
  • Implemented cross-platform score persistence: JSON file on desktop, PlayerPrefs fallback for WebGL
  • Built a reusable settings panel system that works across multiple calling panels without code duplication
  • Deployed to itch.io and made the game publicly playable online

My role: Game Developer

1946 Studio

Photobooth Web App · Team Project · VNR202 · FPT University

A fully browser-based photobooth application themed around Vietnam's National Resistance War of 1946, built by a team of 2 for the VNR202 course at FPT University to commemorate the 79th anniversary of the event (19/12/1946 — 19/12/2025). The app runs entirely in the browser with no back-end — users take webcam photos, apply historical frames, add AI-overlaid accessories and stickers, and download the final composite image as a PNG.

My role: Front-End Developer

Responsible for the following components:

  • 4-Step Flow State Machine — built the app as a step-by-step flow (Select Type → Capture → Select Frame → Result) managed entirely in React state
  • Camera Component — real-time webcam feed using getUserMedia, 3-second countdown, flash effect, and per-photo capture with delete support
  • AI Accessory Overlay — integrated MediaPipe FaceMesh to detect facial landmarks in real-time and auto-place a soldier hat and scarf on detected faces; accessories are baked into the captured photo
  • Frame Preview with Sticker Canvas — live frame preview with drag-and-drop sticker placement using Fabric.js; stickers are composited into the final export
  • Canvas Compositing & Export — final image assembled on HTML5 Canvas (photos + frame + stickers) and exported as a high-quality PNG download

Key achievements and learnings:

  • Built a fully functional AI-powered photobooth that runs entirely in the browser with no server
  • Learned and integrated MediaPipe FaceMesh for real-time face landmark detection and accessory placement
  • Gained hands-on experience with Fabric.js for interactive canvas manipulation (drag, scale, rotate stickers)
  • Deepened understanding of the HTML5 Canvas API for multi-layer image compositing and PNG export
  • Deployed to Vercel and used by students at the VNR202 event to commemorate the 79th anniversary
